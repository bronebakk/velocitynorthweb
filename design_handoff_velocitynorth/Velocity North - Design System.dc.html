// 00 — overview

The numbers were never the hard part.

A single source of truth for velocitynorth.ai — consolidated tokens, component specs with every state, per-screen layouts, accessibility rules, and a sequenced build order. Everything here is expressible as CSS rules, small HTML fragments, and shortcode/CPT output inside GeneratePress + Code Snippets. No custom theme, no build step.

The positioning

Ad platforms under-report conversions (consent, ITP, ATT). The dashboards lie. Velocity North reconciles every conversion server-side and optimises on real profitability. The design must feel like measured truth: precise, technical, no fluff.

The motif

A developer-terminal language on a warm light page — > prompts, // comment eyebrows, a blinking red cursor, .exe window chrome. Selected data moments stay dark consoles even in light mode.

// 07 — accessibility

Baked in, not bolted on

Focus

Visible :focus-visible everywhere — 2px --text outline, 2px offset. Skip-to-content link. Logical tab order; drawer traps focus + restores on close.

Contrast

Body/secondary text pass AA. --dim, --primary, --verified are AA-Large only — gate by size/weight (see §01). CTA text uses bold ≥18.66px or the --primary-strong fill.

ARIA

Nav aria-expanded/aria-controls; disclosure via native <details>; form aria-invalid + aria-describedby + aria-live result. Decorative dots/cursor get aria-hidden.

Reduced motion

@media (prefers-reduced-motion:reduce) → cursor solid, transitions 0ms, no auto-scroll. Motion is decorative only; never conveys state.
